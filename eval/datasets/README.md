# Eval Dataset Provenance

## Overview

The boule eval uses an MT-Bench subset paired with length-confound answer pairs.
MT-Bench provides multi-turn questions and high-quality reference answers across eight categories
(writing, roleplay, reasoning, math, coding, extraction, STEM, humanities).

## Label Reuse Policy

The public human preference labels and GPT-4 preference labels published with MT-Bench are
**reused, not regenerated**. We do not collect new human labels. Our judging procedure is
measured against the existing published labels as ground truth, consistent with the evaluation
design described in spec section 9.

## Open Dependency

The exact availability and file paths of the MT-Bench label files must be confirmed when the
dataset is downloaded. This is an open dependency, it is surfaced here explicitly rather than
hidden. Until the dataset is present locally, the eval harness cannot be run end-to-end.

## Generated Outputs

`eval/results/` is gitignored. All result files are regenerated by the harness on each run and
must not be committed.
